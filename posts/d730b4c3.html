Hexo + NexT 博客 SEO 优化

发表于 2020-12-28 分类于 实践记录 ， Hexo
相比在博客园等网站上写博客，自己搭建个人小站能让我觉得在茫茫互联网上，有了个自己的窝。但搭建好后，被搜索引擎搜索到还比较困难，还需要针对谷歌和百度等主流搜索引擎的抓取做优化，因为 NexT 的更新，一些优化方式有点过时，这里记录一下新版本的优化过程。

前言

博客建好后，在各个搜索引擎搜索 site:blog.locotor.cn，都显示：“找不到和您查询的“site:blog.locotor.cn”相符的内容或信息”。

咱写个博客也不能变成无可循迹的虚无之地呀😥，针对 SEO 的操作开搞！
因为 NexT 几次版本更新后，更改了配置方式，目前不推荐修改包内文件，所以对一些网上的配置方式有些摸不着头脑，这里也记录下新版本的配置途径。

提交注册

要让搜索引擎能够快速找到我的网站，首先需要在搜索引擎的站长工具平台上提交注册，这里记录下百度和谷歌的提交过程。

谷歌

  1. 进入 Google Search Console 提交网站。

  2. 选择资源类型： 提交时需要先选择 网域 或者 网址前缀 类型,。

    这两种类型，收录行为，验证方式，以及支持情况有所不同。
    对于搜索引擎来说，网址前缀不同，完全就是不同的网站。例如以域名 locotor.cn 来说，不同的协议，不同的子域名：

    https://www.locotor.cn
    http://www.locotor.cn
    https://locotor.cn
    http://locotor.cn
    https://blog.locotor.cn
    http://blog.locotor.cn

    都是完全不同的网站，搜索引擎会对它们分别收录，所以也要分别提交注册。
    而网域则不同，会将同一个域名下所有版本的资源都进行整合，不论是 https，还是 http，是否带 www 子域名，全都整合在一起。
    具体区别有：

    网域 网址前缀
    SEO的数据 整合在一起收录 依照不同的网址版本分开收录
    安装和验证方式 只能通过设置 DNS 验证 有多种验证方式
    旧版表单与功能的支持 有些表单不支持 支持所有功能

    目前我只部署了 https://blog.locotor.cn 这一个网址，所以只需选择 网址前缀 这一种方式即可。这两个类型不冲突，后续如果有需要，可以再同时提交网域和网址前缀。

  3. 验证网站拥有权：

    • 域名提供商验证：网域和网址前缀都支持的验证方式，会弹出一个 DNS 记录验证。
      复制这个值，在阿里云万网中添加一个域名解析 TXT 设置，粘贴到记录值那里。如果是选择的网域，主机记录就填 @，如果是选的网址前缀，主机记录就填子域名（例如 blog）。

    • html 文件验证：会弹出一个 html 文件链接，将文件下载并放在网址根目录。
      但是将下的文件放入 Hexo 的 source 文件夹是无效的，因为 Hexo 在编译后，会将它的内容提取并嵌入 Hexo 的 html 页面模板中，需要通过配置 config.yml > exclude 来剔除该文件。

    • html 标记验证：需要在网站 html 文件的 <head> 标签中加入一行 Search Console 弹出的 <mate name="google-site-verification" content="XXXXX">。
      在 NexT 的配置文件中配置 google_site_verification: XXXXX

    • Google Analytics：如果拥有 Google Analytics 账户，并且首页对应的包含 analytics.js，就无需操作，直接就可以点验证了。
      对于使用谷歌网站分析工具的站长来说最快捷。

    因为我可以给阿里云万网添加 TXT 记录，并且它是唯一支持两种类型的验证方式，所以选择 DNS 验证对我来说最方便，果断选之！😆

  4. 添加站点地图：使用 npm install hexo-generator-sitemap 安装 sitemap 插件，无需配置，编译生成网址时会自动创建 sitemap.xml 文件到根目录。

SEO 优化

链接持久化

由于 Hexo 的默认链接配置是 permalink: :year/:month/:day/:title 这种类型，所以最终会生成如 https://blog.locotor.cn/2020/1/1/测试博文 这种路径来。
这种默认路径一来路径层级过多（一般搜索引擎常爬取前三层），二来会出现中文链接，都不利于 SEO。解决方案如下：

  1. 给每个文章添加 urlname 属性，用于输入文章的英文标题，然后再修改 _config.yml： 这样生成的路径为 https://blog.locotor.cn/posts/20200101-test-post.html（.html 后缀会更像静态网站，利于 SEO）。如果文章内没写上英文名，默认用“unnamed-post”代替。
  2. 如果觉得每次要额外写个英文名太麻烦，可以安装 hexo-abbrlink ： npm install hexo-abbrlink，然后修改 _config.yml： 这会给每个文章自动创建一个唯一 ID 标识，文章路径都不会变了

NexT SEO 配置

NexT 主题提供了 SEO 优化配置项。按NexT官方文档描述，主要提供了下列这些配置项。

  • 配置 index_with_subtitle: true 可以在首页添加副标题信息。
  • 提供 google、bing、yandex 和 baidu 的验证字符串配置，以便提交 sitemap 文件。
  • 配置 baidu_push:true 来自动推送给百度搜索引擎。

关键词

压缩文件

使用 gulp 进行压缩接口提高访问速度并且降低内存。

robots.txt

搜索引擎的爬虫会以预定的“准则”进入网站，以获取要爬取的页面数（根据网站的权限/大小/声誉得出的抓取页面数），SEO 将此称为抓取预算。
这就意味着，如果阻止网站的某些部分的抓取，搜索蜘蛛就会抓取其他部分来补充页面数。所以阻止爬取没有必要的路径，是非常有益的。
