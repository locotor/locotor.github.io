Next.js+Koa 构建服务端渲染应用

发表于 2021-06-20
Next.js 框架提供了便利的 SSR 应用方案，服务器端渲染、自动代码切分、简单的客户端路由以及集成 webpack，babel等开箱即用的开发工具。但是它默认集成的服务端功能对于复杂的项目，会显得不太够用，所以可以选择与 Express，Koa 等 Node.js 服务器框架集成。这里实践从头搭建一个 Next.js + Koa 的服务端渲染应用

Next.js 自带一个简单的网络服务器，默认监听 3000 端口，可以根据 URL 路由，将 pages 文件夹中的 React 组件渲染为 HTML 返回，或是 js 请求，返回到客户端渲染。但是还不能处理其他数据请求，或是服务端 session 状态，数据接口，连接数据库等功能。所以可以通过集成 koa，将 Next.js 作为它的中间件来运行。koa 具有轻量，便于扩展的特性，可以补足服务器功能不足的同时，扩展更多的场景和功能。

这里从头搭建，看看它们的整合过程，便于更好的理解项目的结构和功能分布。

创建 Next.js 项目

项目可以通过 create-next-app 脚手架搭建：npx create-next-app custom-app-name。它会引入必要的依赖（react、react-dom、next）并创建运行项目所需的脚本命令，创建文件夹（pages文件夹存放页面组件、components文件夹放置通用组件、static文件夹放置静态资源）。然后在终端运行：npm run dev，打开 localhost:3000 就可以看到项目成功运行了。
