比特数图前端新人技术手册（一）

发表于 2021-05-07
本篇先作为索引，列出公司所用的技术栈和相关资源，方便新同学快速上手。

前端基础

首当其冲的就是前端基础开发工具，包括开发语言、开发框架、样式语言等。

开发语言

公司所有的项目，都使用了 TypeScript 进行开发。顾名思义，它就是 Type（类型）+ JavaScript，也就是在 JavaScript 的基础上，添加了额外的静态类型检查。
虽然刚开始使用它的时候，会感觉带上了枷锁跳舞，但是有了它的帮助，可以避免开发过程中绝大多数的类型错误。

要学习 TS，看它的官网是最快的途径。需要注意的是，不要去看 “TypeScript 中文网”，它是爱好者自制的，大量的机翻，会让你困惑不已。
并且，它已经很早就停止了更新，一直停留在了 3.1 版本。而真正的 “微软官方网站“，它是全英文的，后续可能会有中文，不过建议
依旧以英文版为主。不要被吓到，这才只是英文文档的开始，后面还有海量的英文文档（学好英语，别用机翻）。

作为入门，只需要看4个部分：

  1. 快速开始：需要注意的是，这个快速开始很贴心的按不同的学习者分类设计了文档（ 初学者、JS开发者、Java/C#开发者、函数式编程开发者），可以在左边导航栏选择，相信对于大多数同学来说，看 JS开发者篇即可。
  2. 工具快速了解: 带你安装必要的 npm 依赖包和命令行工具，可以在这一步跟着教程，初步的体验这个开发语言。
  3. 基础概念: 可以快速的了解这门语言的所有基本概念，看完这一篇，你就可以说熟悉了这个语言的特点了。
  4. 常用类型：最后一篇，会带你学完所有日常中最常用到的类型，涵盖了你入职开发工作中，所有可能遇到的类型。

看吧，一点都不多，如果英语好的同学，不到2小时就可以看完了，之后，这个语言就不会是你开发路上的拦路虎了。

对于好奇心旺盛的同学，可能跳着看这么少一点，会让他觉得不安，那么这里也介绍一下官方文档中的其他部分，以及为何跳过：

  1. class：因为在我们的开发中，框架会限定对 class 的使用，并且也不推荐在组件中使用类继承。用到最多的是公共服务类的开发，这不算是新人的初始工作。
  2. 复合类型或者高级类型：这在实际开发中遇到的情况不多，也容易加大新人的学习压力，初期的学习，当然以减轻学习曲线为主。
  3. 模块类型定义：新同学通常作为类型的消费者，而不是开发者，并不需要在初期学习庞杂的类型和模块定义系统。

可以看到，略过的部分，并非不看，而是初期用不上。从实用角度来说，应该减少初期学习压力，在后续熟悉开发之后，再回过头继续精进。

开发框架

公司的主力前端框架是 Angular，相比其他两个（Vue 和 React）这个框架非常大而全，其他两个框架主要负责视图部分，其他的开发所需功能（路由、状态管理、脚手架工具、动画、测试、网络请求等等）都依赖社区开源包。而 ng 都是整套自带的，所以刚开始上手可能会有些无所适从。

不过大而全的框架的好处就是设计思想非常完整和统一，也不用在多个开源方案选型之间反复纠结，通过掌握它，相信你也会对前端的工程化有更好的认识。

Angular 的 “中文网” 确实是官方维护的，进度也与英文版本同步，所以可以放心浏览。因为内容庞大，所以部分也是机翻，好在它提供了中英文对照功能，只需要点击中文段落，对应的英文内容就会出现。

作为入门，只需要看4个部分：

  1. 快速上手：按照它的步骤，跟着一步步完成第一个 Angular app 的创建。相信看完这里，对于怎么使用官方的脚手架，已经用框架进行基础开发就已经有了概念了。

  2. 主要概念：组件、模板、指令、依赖注入是这个框架的四个主要概念，这里需要细读一下，通过这一章，可以让你了解如何使用 ng 实现组件化、生命周期、组件传值、模板绑定语法、业务注入等和其他框架同样重要的概念。
    需要注意的是，这一章很多地方讲解过于深入，不利于初学者上手，建议跳过以下部分：

    • 组件章节：动态组件（深入概念，后续精进可以看我其他文章）、Angular 元素（概念性知识，不影响初期开发）；
    • 模板章节：管道（使用不多，可以用函数代替，不用初期学习）、输入输出（和组件章节重复）、模板中的SVG（项目中不常用到）；
    • 指令章节：结构型指令（最常用的是内置的结构型指令，初期开发不会自定义结构型指令）；
    • 依赖注入章节：这是 ng 比较独特的特性，初期不用看多级注入。

  3. 内置特性：前面看了 ng 的视图库部分，它们和 vue、react 没有本质区别。这章主要看看它内置的其他库，只需要看 HTTP客户端（类似 axios）、表单（有简单的模板驱动表单和细致控制的响应式表单）、路由

  4. 参考手册：只需要看 Observables&RxJS 部分，因为 Observable 是 Angular 的基本异步和数据源方案，所以需要通过这一章了解它的基本使用。

看起来这会比其他框架要学的内容多一些，其实你等于是看了 vue + vue-router + axios + vuex 等，学习总量是类似的，Angular 框架的就是谷歌提供了整套解决方案。

其他工具

除了基本的语言和前端框架，开发中还需要用到其他的一些工具：

  • zorro 组件库：这是阿里巴巴开发的一套成熟的基础组件库。几乎可以涵盖日常开发的所有基础组件，不需要通看浏览，在实际开发里，根据需求去查找是否有这个组件，然后再按照示例去使用即可。
  • scss: 常用的增强样式语法，可以简化 css 的开发，减少出错。不用通看，只需要了解嵌套、变量、Mixin、对象、以及各种内置的方法即可
