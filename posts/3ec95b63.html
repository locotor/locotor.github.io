Hexo 搭建个人博客到 GitHub 和 Coding

发表于 2020-12-21 分类于 实践记录 ， Hexo
最近在同事的安利下尝试了 Hexo 搭建博客。 博客站点双线部署到 Github 和 Coding，配置DNS的时候让国内的线路流量到 Coding 上，其他的线路流量到 GitHub 上，解决了百度不能搜录 Github Pages 网站的问题。实现双线部署，支持 HTTPS 协议访问。折腾了一阵，这里记录一下搭建过程。

创建 Hexo 项目

项目需要 NodeJs 和 Git 环境。这里默认环境已经装好了，怎么装 NodeJs 环境，或是使用 Git，教程很多，就不累述了。

安装 Hexo

先全局安装 hexo-cli: $ npm install -g hexo-cli 然后就可以通过 $ hexo <命令> 来调用。
如果是在项目范围内安装，$ npm install hexo-cli 通过 $ npx hexo <命令> 调用。

创建项目

安装好 Heox 后，就可以直接创建博客项目：

项目做需要的文件和目录结构都会被自动创建好。

本地新建博客和预览

通过 $ hexo server 命令即可在本地发布博客网站，默认的访问路径和端口是 localhost:4000。这个本地服务器会监测内容变化，实时更新博客。
用 $ hexo new <文章名> 即可创建一篇新的博客，刷新浏览器就可以看到新创建的空白博客。
剩下的，就是部署到互联网上了。

部署到 Github 和 Coding

将 Hexo 项目提交到 Github 仓库，然后在 Github 和 Coding 分别创建一个承载 Hexo 生成内容的仓库，彼此同步，用以生成 Pages 网站。

Coding

  1. 创建仓库：Coding 改版后是以“团队/项目/代码仓库”的结构来创建，所以需要先创建一个 blog 项目，以后给朋友建个博客也放这个项目里吧。然后在这个项目中创建一个 locotor 仓库。

  2. 开启 Pages 服务：

    1. 在左下角”项目设置/功能开关”中开启“持续集成”和“持续部署”。
    2. 回到项目首页，在新增的持续部署侧边栏选项中，选择“静态网站”。
    3. 新建网站选择刚刚创建的仓库，网站类型选择“静态网站”。

现在 Coding Pages 就创建好可以直接访问了，每次 loctor 仓库有内容更新，这个博客网站内容也会刷新。
那么内容从哪里来呢？就要等待 Github 的仓库自动同步了。

GitHub

在 GitHub 上创建两个仓库，一个用来维护 Hexo 项目和博文文件，一个用来做博客网站的存储仓库并提供 Github Pages 服务。

为了后续同步，先新建一对 ssh-key：$ ssh-keygen -t rsa -C "yourName@example.com" ，这个命令会在电脑的 C/Users/username/.ssh 文件夹下生成两个文件（id_rsa 和 id_rsa.pub），分布是私钥和公钥，后面需要用到它们，备用。

Pages 仓库

  1. 创建一个名为 locotor.github.io 的仓库，因为要方便 Coding 同步，仓库权限设为 public。勾选创建默认的 index.html,README.md，不然在空仓库状态下还不能开通 Pages 服务。
    后续 Hexo 生成的博客静态网站文件，将自动上传到这个仓库。

  2. 在仓库的 Settings > Options > GitHub Pages 下配置 Source 为 Branch:master/(root)。此时通过访问 https://locotor.github.io 就能看到默认创建的 index.html 的内容。

  3. 在仓库的 Settings > Deploy keys 下新建一个 Deploy key，这个 key 的内容是刚刚创建的 ssh-key 的 public 部分，用记事本打开复制粘贴过来即可。名字可以任意设置，但是需要勾选 write access(写入权限)。

  4. 在仓库的 Actions 中创建一个新的自动脚本，功能是每次这个仓库的内容更新后，同步内容到 Coding 的仓库，这里使用了 yehan-coding/yehan-git-sync 来同步代码：

    Hexo 仓库

  5. 创建一个名为 blog-builder 的仓库，仓库权限可以设为 private。

  6. 在本地刚刚创建的 HexoExample 路径下，使用 git init 命令初始化项目的 Git。然后使用命令 git remote add origin https://github.com/locotor/blog-builder.git 和 blog-builder 仓库建立连接。

这里我希望把博客部署在我自己的二级域名：blog.locotor.cn，

自定义域名

目前已经能够通过 locotor.github.com 和 coding-pages-bucket-1575455-8137703-8340-424565-1303365836.cos-website.ap-beijing.myqcloud.com/ 这两个路径来访问发布的博客。
但是我想将它们自定义为 blog.locotor.cn。 并且能够根据用户线路自动解析。
首先要修改 Hexo 项目根目录的配置文件 _config.yml：

接着分别在 GitHub 和 Coding 上配置一下。

  • GitHub：只需要在 Hexo 项目的 source 文件夹内，创建一个名为 CNAME 的文件，文件内写入 blog.locotor.cn。提交到 GitHub 后，就会自动识别。
  • Coding：在 持续部署 > 静态网站 > 自定义域名 里，新建域名，在弹出的弹窗中填入 blog.locotor.cn 绑定后，网站会自动重新部署（如果域名未备案，会部署失败），并生成一个 CNAME 指向（blog.locotor.cn.cdn.dnsv1.com） 后续会用到。
    同时会去申请 SSL 证书，申请成功后会通知你重新部署一次。

添加解析记录

自定义域名后，需要手动去对应的域名托管商添加两个解析记录。这里使用的是阿里云万网。在 域名解析 > 解析设置 中添加记录，类型均为 CNAME，主机记录为 blog，记录值分别是：

  • GitHub：locotor.github.com
  • Coding： blog.locotor.cn.cdn.dnsv1.com

国内备案和审批

要给部署在 Coding 国内服务器的博客自定义域名，首先需要ICP备案域名，否则会部署失败，备案成功后再去公安联网备案，然后在腾讯云申请 SSL 证书，实现和Github 的双线部署，全站 HTTPS 协议访问。

  1. ICP 备案：域名在阿里云万网购买，可以通过阿里云备案服务号备案，填写信息后，手机视频认证，短信认证，然后大约需要等待一周左右。然后将审批通过的备案号放置在网站的 footer 处。
    要插入备案号，需要先解除掉 NexT 配置文件 > custom_file_path > footer 的注释，在 source > data 目录中创建 footer.njk 文件，插入备案号链接 <a href="http://beian.miit.gov.cn/" rel="nofollow" id="icp-code"> 渝ICP备2020014938号</a>。
  2. 公安联网备案：ICP 备案成功后，按邮件提示，完成公安联网备案，按相同的方法，插入公安备案号链接。
  3. 申请 SSL 证书：在 Coding Pages 的自定义域名页面，点击验证网站，按提示在万网域名解析中插入一个 TXT 记录值，即可完成验证。
    证书申请成功后，即可直接一键部署腾讯云托管的证书到 blog.locotor.cn 下，完成部署。
